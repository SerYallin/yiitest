Данный проект предназначен оценить навыки разработки на PHP Yii2, включая работу с ActiveRecord, AJAX, валидациями, загрузкой файлов и структурой кода.
### Описание
Реализовать web-приложение для учёта заявок на обслуживание в офисе.
### Функциональность
1. Список заявок
   - Таблица с заявками: ID, Заголовок, Статус, Дата создания, Действия
   - AJAX:
   - Изменение статуса заявки (new, in_progress, done) через выпадающий список
   - Удаление заявки
   - Кнопка "Создать заявку" — открывает форму (можно в модальном окне)
2. Создание заявки
   - Форма с полями:
   - Заголовок (обязательное)
   - Описание (опционально)
   - охраняется с status = new, текущей датой в created_at, updated_at
3. Просмотр заявки
   - Страница с полными данными заявки
   - Список комментариев
   - AJAX-форма добавления комментария
4. Комментарии к заявке
   - Добавляются к заявке
   - Каждый комментарий хранит текст, дату создания
   - Комментарии отображаются в порядке создания
5. Загрузка изображений
   - Каждая заявка может содержать до 5 изображений
   - Поддержка форматов: JPG, PNG
   - Размер одного файла — до 2MB
   - Изображения загружаются через AJAX, без перезагрузки
   - После загрузки — отображаются превью
   - Возможность удалить изображение (тоже AJAX)

### Требования
1. Использовать Yii2 (basic или advanced) расширения запрещены
2. Использовать ActiveRecord
3. Использовать AJAX библиотеки jQuery и другие вспомогательные JS библиотеки разрешены для:
   - создания заявки
   - добавления комментариев
   - изменения статуса
   - загрузки и удаления изображений
4. Защита от CSRF
5. Валидации на клиенте и сервере
6. Уникальные имена загружаемых файлов
7. Хранить изображения в /uploads/request/{id}/filename.jpg


## Выполнено.

1. Проект выполнен на PHP используя фреймворк **YII2**, с использованием модульного подхода.
2. Установка выполнена с использованием базового пакета(**YII2 base**).
3. База данных используется **PostgreSQL**, поэтому большая просьба при тестировании использовать ее, так как в MySQL есть небольшие расхождения, в частности невозможно будет импортировать схему из-за использования поля типа массив чисел. Также есть небольшие расхождения в написании тригера для БД.
4. Для работы с базой данных используется **ActiveRecord**.
5. Для работы с формой используется **ActiveForm**.
6. Для работы с Ajax формами(работа без перезагрузки) используется встроенная библиотека **jQuery.pjax**, и **JQuery.ajax**.
7. Валидация на стороне клиента выполнена средствами фреймворка **YII2 ActiveRecord**
8. Валидация на стороне клиента выполнена средствами **ActiveRecord** с использованием пользовательских валидаций, где это требуется.
8. Защита от CSRF выполнено средствами фреймворка (подключено в настройках `configs/web.php` с опрокидыванием ключа в запросы через `js` библиотеку `yii.getCsrfToken()`)
9. Файлы картинок загружаются в указанный требованиями путь. Если картинка с таким именем существует, к имени добавляется счетчик, гарантируя таким образом уникальность имен файлов.

Установка
------------

### Установка с Composer

1. Установить или убедится что установлен [composer](https://getcomposer.org/download/) 

2. Скопировать файлы с гита вручную либо коммандой `git clone`
~~~
git clone git@github.com:SerYallin/yiitest.git
~~~
3. Выполнить установку зависимостей
~~~
composer install
~~~
4. Настроить подключение к базе данных в файле `configs/bd.php`
5. Установить схему базы данных.
~~~
./yii migrate/up --migration-path modules/requests/migrations
~~~

6. Не забудьте убедиться, что развернута локальная среда, это может быть `docker`, `LAMP(c PostgreSQL)`, `ddev` или любая другая.
